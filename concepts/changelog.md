# WTF is a Changelog?

A file (usually `CHANGELOG.md`) that lists notable changes for each version of a project. It tells users what was added, fixed, or removed without reading every commit.

```
# Changelog

## [2.1.0] - 2024-03-15
### Added
- OAuth2 support

### Fixed
- Null pointer in API handler

## [2.0.0] - 2024-01-10
### Removed
- Legacy v1 endpoints (BREAKING)
```

**Examples:** [keep-a-changelog](https://keepachangelog.com/), [conventional-changelog](https://github.com/conventional-changelog/conventional-changelog), [auto-changelog](https://github.com/cookpete/auto-changelog)
