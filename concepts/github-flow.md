# WTF is GitHub Flow?

A simplified branching model: branch off main, make commits, open a pull request, get reviewed, merge back to main. That's it. Designed for teams that deploy frequently.

```
main:     ──●───●───●───────●───●──
              \             /
feature:       ●───●───●───
                    |
                 Open PR
                 Review
                 Merge
```

**Examples:** [GitHub Flow Docs](https://docs.github.com/en/get-started/using-github/github-flow), [GitHub Guides](https://guides.github.com/introduction/flow/), [Atlassian Comparison](https://www.atlassian.com/git/tutorials/comparing-workflows)
