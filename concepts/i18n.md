# WTF is Internationalization (i18n)?

Designing your app so it can be adapted to different languages and regions without code changes. You externalize all user-facing strings, handle date/number/currency formats, and support right-to-left text. The "18" in "i18n" is the number of letters between "i" and "n." The actual translation work is localization (l10n).

```
Hardcoded:                    Internationalized:

<h1>Welcome</h1>             <h1>{t('welcome')}</h1>

                              en.json: { "welcome": "Welcome" }
                              pt.json: { "welcome": "Bem-vindo" }
                              ar.json: { "welcome": "مرحبا" }
```

**Examples:** [i18next](https://github.com/i18next/i18next), [FormatJS](https://github.com/formatjs/formatjs), [vue-i18n](https://github.com/intlify/vue-i18n)
