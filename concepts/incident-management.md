# WTF is Incident Management?

The process of detecting, responding to, and resolving production outages or degradations. It includes the alert, triage, communication, fix, and post-mortem. Good incident management has clear roles (incident commander, communicator), a war room, and a blameless review afterward.

```
  Detection → Triage → Response → Resolution → Post-mortem

  Roles during incident:
  ┌─────────────────────────────────────┐
  │ Incident Commander: coordinates     │
  │ Communicator: updates stakeholders  │
  │ Responders: fix the problem         │
  └─────────────────────────────────────┘

  After: blameless post-mortem → action items
```

**Examples:** [PagerDuty](https://www.pagerduty.com), [incident.io](https://incident.io), [Grafana OnCall](https://github.com/grafana/oncall)
